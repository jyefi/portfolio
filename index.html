Master Thesis

The project used for my master's thesis consists of the implementation of a system distributed in different components, among which are:

  • A relational data persistence layer built on top of PostgreSQL
  • A non-relational data layer implemented in MongoDB
  • A relational data access layer (Python / Django)
  • A non-relational data access layer (Java / Spring Boot)
  • A data access gateway (Python / Django)
  • A presentation layer to be developed with React Native (expo)


All of the above orchestrated with docker-compose, and implementated with secure requests (JWT)

* Work in process

Signal meter with Geolocation

System developed in Python / Flask, for Raspberry Pi (IoT) devices. The system communicates, through AT commands, the signal query to the SIM7600E chip, and in this way captures the signals and stores them in JSON data structures. Finally, the Google Maps API was used to render the signs and their geolocation.

File synchronization system over 4G networks

This component is part of the development carried out for the SENSOPLAG project, which was sponsored by the European regional development fund. The developed component allows automating the tasks of collection, transmission and verification of files transmitted through the rsync protocol. Finally, the synchronized files were transmitted over the Internet through the use of the SIM7600E chip.

Valenbike App

  • Access control (login)
  • Public data consumption (Rest API)
  • Search for stops by name/address
  • Search for stops by geolocation
  • Detail of bicycles / parking spaces available
  • QR code reading


Developed with the talented developer

Automated Notification System

This project was designed to improve the experience with customers, using email marketing and operational messages sent by email channel.

    Some of the big numbers:
  • We sent more than 200M messages by the platform
  • We implement more than 100 of automatic deliver processes (Birthday, payments, etc.)
  • We got more than 90% of clean email data
  • We had a very slow rates of bounced emails

All od these, was made using Talend integration processes, and Apache Airflow, with PostgreSQL Databases and a email service provided by API REST integration.

Customer Datamart

We carry out all the integrations for the integration of Customer Datamart. For that work we developed more than 1K of ETL based on Pentaho Data Integration. We use MySQL databases for combined data, Oracle database sources, and put the data into SQL Server Analysys Server. Finally, we developed visualizations for customer segmentation (Tableau / Qlik Sense), which were reused in the automated communication project.
